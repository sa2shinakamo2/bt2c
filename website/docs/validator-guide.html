Prerequisites

Hardware Requirements

  • • 4 CPU cores
  • • 8GB RAM
  • • 100GB SSD storage
  • • 10Mbps+ internet connection

Software Requirements

  • • Ubuntu 20.04 LTS (recommended)
  • • Docker
  • • Git

Installation

1. Update System

sudo apt update && sudo apt upgrade -y
sudo apt install -y curl git build-essential

2. Install Docker

curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER

Log out and back in for the group changes to take effect.

3. Clone Repository

git clone https://github.com/sa2shinakamo2/bt2c.git
cd bt2c
cp env.template .env

Configuration

Environment Setup

Edit your .env file with the following settings:

# Node Configuration
NODE_NAME=your-node-name
NODE_PORT=3000
NODE_HOST=0.0.0.0

# Network Configuration
NETWORK=mainnet
P2P_PORT=6001

Running Your Node

Start the Node

docker-compose up -d
docker-compose logs -f

⚠️ Important: Your node will need to fully sync with the network before you can start validating. This may take several hours.

Staking

Register as Validator

curl -X POST http://localhost:3000/api/v1/validator/register \
  -H "Content-Type: application/json" \
  -d '{
    "validatorAddress": "your-wallet-address",
    "stake": "1000000000000000000"
  }'

Note: The stake amount is in wei (1 BT2C = 10^18 wei)

Distribution Period Rewards

  • • First node (developer): 100 BT2C
  • • Other validators: 1 BT2C each
  • • Duration: First 2 weeks after launch

Monitoring

Check Node Status

# Health check
curl http://localhost:3000/api/v1/health

# Validator status
curl http://localhost:3000/api/v1/validator/status

# Sync status
curl http://localhost:3000/api/v1/sync/status

View Logs

docker-compose logs -f --tail=100
